# Handoff: Portfolio Redesign — "A Field Guide to a Body of Work"

## Overview

A redesign of the ericjorgensen.com home page. The concept: every project is a
**living specimen** — a small fractal "organism" grown once from a seed, with
handedness (it curls one way) and non-bilateral branching, so the page feels
alive without the layout itself being lopsided. The work is grouped into four
categories arranged as one axis, **machine → hand**, and each category's growth
rule encodes where it sits on that axis. Color is used only as a tiny garnish at
each organism's growth tips. Five square black-and-white portraits sit across the
top as a frontispiece.

## About the Design Files

The files in this bundle are **design references** — an HTML prototype showing
the intended look and behavior, plus a framework-free algorithm. They are **not**
production code to paste in. The task is to **recreate the design inside the
existing client app** (React + Vite + TypeScript + CSS Modules) using its
established patterns. The one piece meant to be used roughly as-is is
`src/growMark.ts` (the mark generator) — it is dependency-free TypeScript.

## Fidelity

**High-fidelity.** Final colors, typography, spacing, and the generative system
are all specified below. Recreate pixel-accurately using the existing stack.

### The four categories (display order: machine → hand)

The array is built hand→machine then **reversed** for display, so sections read
I–IV as: Pure AI → AI-Enhanced → Hand-Written → Pure Creative. Roman numerals are
assigned by display position.

1. **Pure AI Output** — principle "Generated by machine" — accent `#3A5A8C` — style `crystal`
   - PixelWhimsy — "A pixel-art toy for children." (https://pixelwhimsy.com)
   - Big Tiny Games — "Classic games reimagined — a large canvas, original tiny pixel sprites." (https://bigtinygames.com)
   - Budgito — "A personal tool for working against a yearly budget." (https://github.com/nebosite/budgito)
   - QuickNote — "An Android app to capture an idea in two taps."
2. **Hand-Written, AI-Enhanced** — principle "Grown by hand, grafted with machine" — accent `#2E6E6A` — style `grafted`
   - (no items yet — render an italic "No specimens collected yet — this drawer is waiting.")
3. **Hand-Written Code** — principle "Built by hand, untouched by machine" — accent `#5E6B36` — style `sapling`
   - clusterfun.tv — "A Jackbox-style framework, plus the games and apps grown on top of it."
   - hompag — "A dynamic, editable homepage for desktop browsers."
4. **Pure Creative Output** — principle "Grown entirely by hand" — accent `#B5482E` — style `wild`
   - Drawing → route `/art` · Writing → route `/writing` · Photography → route `/photography` (no descriptions)

### The generative marks — `growMark.ts`

- `growMark(seed, style, strokeScale?, growthSeed?)` returns `{ segments, buds, square, cap }`
  in a `0 0 120 150` viewBox, base at (60,148) growing up.
- Per item, seed = the project name; per category emblem, seed = `"<key>-emblem"`.
- `strokeScale`: ~1.18 for big category emblems, ~0.85 for inline item marks.
- `growthSeed`: a global integer; bumping it regrows every organism (was a prototype tweak).
- Render: each `segment` is an SVG `<line>` `stroke="#1d1b16"` `stroke-width=w`
  `stroke-linecap=cap`. Each `bud` is a `<circle r≈2.4*scale+0.6>` (or `<rect>`
  side `4*scale+1.2` when `square`), filled with the category accent (garnish on)
  or `#1d1b16` (garnish off). A reference React renderer is in the file's footer.

## Screens / Views

One page, three switchable layouts (mono tabs top-right; active tab = 2px ink
underline). The prototype defaults to **Field Guide**; the owner especially likes
**Spectrum**. State is a single `layout: 'guide' | 'plates' | 'spectrum'`.

**Shared header**

- Mono kicker (left): "A FIELD GUIDE TO A BODY OF WORK", 11px, letter-spacing .18em, color `#8a8273`.
- Layout tabs (right): Field Guide / Plates / Spectrum.
- `h1` "Eric Jorgensen": Spectral 500, 68px, line-height .98, letter-spacing −.02em.
- Italic tagline: Spectral 300 italic, 21px, color `#4a463d`, max-width 640px.
  Prototype copy: "Things I have made, kept as specimens and arranged in a single
  order — from those grown entirely by the hand to those generated entirely by
  the machine." (Owner's current live tagline is different; confirm which to use.)

**1 · Field Guide** — vertical catalog. Each category is a row: a 128px-wide left
column (roman numeral + a large emblem mark), then a flex-1 body (mono principle
with accent dot · `h2` category name 31px · item rows). Each item row: 46px mark
(left), name (Spectral 500, 20px) + inline "— description" (300, 17px, `#6b6558`),
right-aligned mono plate label `PL. <roman>·<letter>` (`#a39a87`). Items separated
by 1px dashed `#cfc8b8`; sections by 1px solid `#d8d2c4`.

**2 · Plates** — herbarium grid. Per category: a band header (roman + name +
accent dot + right-aligned mono principle, underlined with 1px ink). Then
`grid-template-columns: repeat(auto-fill, minmax(232px, 1fr))`, gap 18px. Each
plate: `#F7F4EC`, 1px `#d8d2c4` border, mark centered (90×120), name, description,
footer with accent dot + plate label.

**3 · Spectrum** — diagrammatic axis. Mono end-labels "The Machine" (left) /
"The Hand" (right) over a 1px ink rule; four equal columns, each with an axis
node dot, a 96×118 emblem, roman + name + principle, then a list of its items
(24px mark + name + small description).

**Frontispiece — portraits** (above the header, all layouts)

- A `display:flex; gap:16px` row of **5** square slots, 120×120 each, 1px ink
  border, `filter: grayscale(1) contrast(1.06)` to force pure B&W.
- Slight organic vertical offsets and rotations per slot (hand-pinned feel):
  | #   | translateY | rotate |
  | --- | ---------- | ------ |
  | 1   | 0px        | −1.1°  |
  | 2   | 2.8px      | 0.9°   |
  | 3   | −1.4px     | −0.7°  |
  | 4   | 1.8px      | 1.3°   |
  | 5   | −0.6px     | −0.9°  |
- Hover: the image lifts `translateY(-5px)`, transition `.4s cubic-bezier(.2,.7,.2,1)`.
- In the prototype these are drop-slots; in the real app they are the existing
  `PortraitStrip` `<img>`s from `/api/portraits` — apply the same grayscale, size,
  border, offsets, rotation, and hover to those. (Distribute the 5 offset/rotation
  pairs by slot index; the existing rotation-every-10–15s behavior stays.)

## Interactions & Behavior

- Layout switch: click a tab → set `layout` state → show that layout, move the
  underline. No animation required.
- Portrait hover lift (above). Existing portrait auto-rotation: keep.
- Visit counter (`POST /api/visit`) and footer: keep from current `HomePage.tsx`.
- Fully responsive isn't specified; the prototype targets desktop (~1180px
  container, 56px side padding). Mobile can stack the spectrum columns and reduce
  portraits to 3 (the existing `MOBILE_BP` logic already does the latter).

## State Management

- `layout` (string enum) — which layout is shown.
- `growthSeed` (int, default 1) — optional; only if you expose mark reseeding.
- `garnish` (bool, default true) — optional; ink-only vs. accent buds.
- Existing: `visitCount`, portrait `pool`/`slots`.
